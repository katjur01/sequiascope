Variant Calling

The Variant Calling module is divided into two submodules: Somatic Variant Calling and Germline Variant Calling. Both submodules allow users to review, filter, and select variants for further analysis and reporting. The module provides interactive tables, filtering options, and visualizations to streamline the review process. Both submodules provide powerful tools for reviewing, filtering, and selecting variants for further exploring and reporting through interactive tables, advanced filtering options, and integrated visualizations.

Overview of Features

Interactive data tables

All variant data are displayed using the reactable package, providing fully interactive tables with real-time responsiveness:

  • Sorting: Click any column header to sort data in ascending or descending order
  • Built-in filtering: Use the search boxes within each column to filter data on-the-fly
  • Resizable columns: Adjust column widths by dragging the column borders
  • Multi-column operations: Combine filters and sorts across multiple columns simultaneously

Tip: Hold the Shift key while clicking column headers to sort by multiple columns simultaneously. To clear sorting, hold Shift and click again — the order cycles through ascending → descending → unsorted.

Filtering Menu

While the interactive table provides basic filtering capabilities, the dedicated filtering menu offers more sophisticated filtering options for parameters that require specific logic:

  • Gene Region: Filter specifically for exon and splice regions (not possible with standard column filters)
  • Consequence: Select from multiple consequence types
  • Coverage Depth: Set minimum coverage thresholds
  • Population Frequency: Filter by gnomAD allele frequencies
  • ClinVar Significance: (Germline only) Filter variants by clinical significance
  • Custom Column Selection: Show or hide columns to focus on relevant data

::: {.alert .alert-warning} Default Filters Applied: When you first view the data, the following filters are automatically applied to ensure optimal performance and focus on relevant variants:

gnomAD_NFE ≤ 0.01 (rare variants only) coverage_depth > 10 (sufficient read support) Consequence ≠ "synonymous_variant" (exclude silent mutations) gene_region == "exon" OR "splice" (coding regions only) :::

::: {.alert .alert-success} Performance Tip: Apply filters before reviewing or marking variants. Working with a filtered subset significantly reduces data reload times and keeps the application responsive. :::

Variant Selection

The module allows you to curate variants of interest through an intuitive selection workflow:

  • Mark Variants: Select variants directly from the interactive table

  • Clinical Significance:

    • Mark somatic variants as potentially oncogenic
    • Mark germline variants as potentially pathogenic

  • Review Panel: Selected variants appear in a dedicated table for easy review

  • Edit Selection: Remove variants from your selection if needed

  • Report variant: Only selected variants will be included in the final report

  • IGV Integration: Only selected variants can be inspected in the Integrated Genome Viewer

Data Export

Export your variant data in multiple formats for downstream analysis or reporting:

  • Supported Formats: CSV, TSV, XLSX
  • Export Options:
    • Export the currently filtered table (only visible variants)
    • Export the complete dataset (all variants)

Integrated Genome Viewer (IGV)

Validate variants at the read level with seamless IGV integration:

  • Multi-Patient Analysis: Select multiple patients simultaneously for comparative viewing
  • Direct Launch: Open IGV directly from the module with variants pre-loaded
  • Read-Level Validation: Inspect sequencing reads, coverage, and alignment quality
  • Context Visualization: View variants in their genomic context with annotations

Submodule-Specific Features

Somatic Variant Calling

The somatic submodule is designed for analyzing tumor-specific variants with specialized metrics and visualizations:

Key Features:

  • Tumor Metrics: Variant allele frequency (VAF) and tumor sequencing depth
  • Cancer Databases: Filter by COSMIC and CGC_Somatic annotations

Tumor Variant Frequency Histogram

This histogram displays the distribution of tumor variant allele frequencies (VAF) across all somatic variants. Selected variants are marked as blue lines.

Sankey Diagram

The Sankey diagram visualizes the relationships between somatic variants, their associated genes, and the biological pathways in which these genes participate. This integrative view highlights how molecular alterations connect to functional contexts, offering insights into pathway-level disruptions.

Germline Variant Calling

The germline submodule focuses on hereditary variants with clinical interpretation tools: Key Features:

  • Tumor Metrics: Variant allele frequency (VAF) and tumor sequencing depth
  • Clinical Annotations: ClinVar significance
  • Hereditary Cancer Genes: Filter by CGC_Germline and TruSight Genes panels

Table Columns

The following table describes the columns available in the interactive tables for both submodules:

Column Description Somatic Germline
Gene_symbol The gene associated with the variant. ✓ ✓
var_name The name of the variant. ✓ ✓
HGVSc HGVS coding sequence annotation. ✓ ✓
HGVSp HGVS protein sequence annotation. ✓ ✓
variant_type The type of variant (e.g., SNP, insertion, deletion). ✓ ✓
tumor_variant_freq The variant allele frequency in the tumor sample. ✓
tumor_depth The sequencing depth at the variant position in the tumor sample. ✓
variant_freq The variant allele frequency in the germline sample. ✓
coverage_depth The sequencing depth at the variant position in the germline sample. ✓
gnomAD_NFE The population frequency of the variant in the gnomAD Non-Finnish European population. ✓ ✓
Consequence The predicted functional consequence of the variant (e.g., missense, synonymous). ✓ ✓
clinvar_sig The clinical significance of the variant according to ClinVar (e.g., pathogenic, likely pathogenic). ✓
COSMIC Indicates whether the variant is listed in the COSMIC database. ✓
CGC_Somatic Indicates whether the variant is associated with cancer in the CGC database. ✓
CGC_Germline Indicates whether the variant is associated with hereditary cancer in the CGC database. ✓
TruSight Genes Indicates whether the variant is included in the TruSight Cancer Panel. ✓

Note: The application recognizes additional columns beyond the required ones. If your dataset includes extra columns, they will still appear in the table and can be used for filtering or display. However, these columns will not have customized labels — only minor formatting is applied (dots and underscores are replaced with spaces).

Limitations

  • The IGV integration requires BAM/BAI files to be available for the selected patient.
